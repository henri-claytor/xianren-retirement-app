## Context

生產版設計決策（本次 Prototype 同步對齊）：
- **市場**：由 API 依代號自動判斷，不需用戶選擇
- **現價**：由 API 即時取得，不需用戶輸入
- **債券比例 %**：由 API 提供，計算三桶金時自動換算

因此用戶只需輸入：**代號/名稱（搜尋）、數量（股數/單位數）、成本（成本均價/成本淨值）**

## Goals / Non-Goals

**Goals:**
- 個股/ETF 卡片移除：市場下拉、現價輸入欄
- ETF/基金卡片移除：債券比例 % 輸入欄
- 個股/ETF 加顯示損益金額（NT$），costPrice > 0 才顯示
- 基金加 `costNav` 輸入欄，加顯示損益金額與損益 %

**Non-Goals:**
- 移除資料模型中的對應欄位（保留供未來 API 填入）
- 實作真實 API 串接（Prototype 維持 mock）

## Decisions

**移除欄位後資料行為：**
- 現價（currentPrice）：保留資料欄位，搜尋時仍自動帶入 mock 現價，只是不再顯示輸入框供用戶修改
- 市場（market）：保留欄位，搜尋時自動判斷，不顯示選擇器
- 債券比例（bondRatio）：保留欄位，搜尋時自動帶入 mock bondRatio

**損益金額計算：**
```
個股/ETF：pnlAmt = val - (costPrice × shares × currency換算)
基金：    pnlAmt = (nav - costNav) × units × currency換算
         pnlPct = (nav - costNav) / costNav × 100
```

**精簡後各卡片欄位 + 排版（方案 C）：**

```
左欄（是什麼）    右欄（多少）
[ 搜尋代號/名稱 ] [ 股數      ]
[ 名稱          ] [ 成本均價  ]
────────────────────────────────
市值 X萬   損益 +X萬   +X%
```

左欄 = 識別資訊（搜尋、名稱），右欄 = 數字（數量、成本）。

| 持倉 | 左欄 | 右欄 | 底部顯示 |
|------|------|------|---------|
| 個股 | 搜尋、名稱 | 股數、成本均價 | 市值、損益NT$、損益% |
| ETF  | 搜尋、名稱 | 股數、成本均價 | 市值、損益NT$、損益%、bucket |
| 基金 | 搜尋、幣別 | 單位數、成本淨值 | 市值、損益NT$、損益% |

## Risks / Trade-offs

- [移除現價輸入] → mock 搜尋帶入的現價無法手動修改，Prototype 展示影響微小，生產版由 API 取代
- [移除債券比例輸入] → ETF/基金 bucket 分類仍以 mock 的 bondRatio 值計算，搜尋未選到的持倉 bondRatio = 0 → 歸長期桶（合理預設）
