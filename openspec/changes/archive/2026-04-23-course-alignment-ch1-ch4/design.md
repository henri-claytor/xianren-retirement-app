## Context

App 目前對齊 v1 課程結構。嫺人 v2 課程出現 CH1~CH4 四個結構性落差，但原規劃的「S0 願景問卷 + 風險屬性問卷」對退休族（App 主要受眾）門檻過高，每多一題就流失一批用戶。本 change 改採「零問卷」路線：利用 S1 既有資料自動推斷情境，用一鍵三選取代問卷。CH5/CH6 另案處理。

## Goals / Non-Goals

**Goals:**
- CH1~CH4 每章都有對應的 App 區塊可操作或參考
- **用戶動作增量 ≤ 1 次點擊**（A3 風險屬性選擇）
- 維持 Dark Theme 與既有 Accordion / VerdictCard 模式
- 所有新區塊從既有資料自動推算，不新增輸入欄位（除風險屬性外）

**Non-Goals:**
- CH5（家庭、保險、傳承）與 CH6（非財務）本次不做
- 不新增獨立問卷頁或 S0 願景頁
- 不引入新狀態庫
- 不做 Android 同步
- 不改 Tab Bar 結構

## Decisions

### D1：CH1 用「自動生成摘要」取代 S0 問卷頁
- Dashboard 頂端渲染一段自然語言情境摘要，資料源全部來自 S1：
  > 「你目前 50 歲，距離退休 15 年，預計退休後每月花 8 萬。依此試算，你需要每月存 3.5 萬才能達標。」
- 用戶想調整時點段落右側「編輯」icon，展開小型 Accordion 可改「退休年齡」「退休月支出」兩欄（這兩欄本來就在 S1，此為快捷入口）
- **為什麼**：S1 已有資料，再讓用戶填 4 題重複且摩擦大；自然語言摘要比問卷更有「看見自己」的感覺
- **替代**：S0 獨立頁 → 放棄，動作太多

### D2：CH3 用「一鍵三選」取代風險問卷
- A3 頁首放三張橫向大卡片：🛡️ 保守 / ⚖️ 穩健 / 🚀 積極
- 每張卡片顯示描述 + 建議股債比，點擊即套用到 A3 配置並儲存 `riskProfile`
- **為什麼**：5 題問卷 → 1 次點擊，心智負擔最低且結果一樣
- **替代**：5 題評分問卷 → 放棄，過度工程

### D3：A2 三風險分項採「單因子敏感度」
- 以基準情境為錨，逐一放大單一因子（通膨 +1%、壽命 +5 年、醫療月支出 +20%），顯示新成功率與 Δ
- 最負 Δ 卡片紅框「最需留意」
- 附註「單因子分析，實際風險會交互作用」
- **為什麼**：用戶看得懂哪個風險最吃成功率；零用戶輸入

### D4：B3 採「規則推薦」
- 三條規則卡片（市場跌幅、短桶警戒、年度再平衡），顯示目前狀態與建議動作
- 短桶狀態自動從 S2 資料算：`短桶金額 / 月支出` 月數
- 不做自動執行，純教育
- **為什麼**：信任度與稅務議題，教育優先

### D5：`riskProfile` 是唯一新增 store 欄位
- 不新增 `vision`（用 S1 既有資料即時推算即可）
- `riskProfile: 'conservative' | 'balanced' | 'aggressive' | null`
- 未選過時 A3 顯示預設「穩健 50/50」但在三張卡片上方提示「點選一項更貼近你的屬性」

## Risks / Trade-offs

- **風險**：自動生成的摘要措辭不夠準確，用戶感到「被代言」→ **緩解**：措辭保守、只陳述 S1 填過的事實、保留編輯入口
- **風險**：一鍵三選過於簡化，配置不夠個人化 → **緩解**：A3 仍允許手動微調股債比；三選只是預設
- **風險**：A2 單因子忽略交互作用 → **緩解**：頁面附註
- **風險**：B3 再平衡規則理想化 → **緩解**：頁面標註稅務/手續費免責

## Migration Plan

- `useFinanceStore` 新增 `riskProfile` 欄位（default null），不破壞既有資料
- 部署前本地驗證 Dashboard 摘要、A3 三選、A2 三風險、B3 四個區塊
- `npx vercel --prod` 從 repo root 部署

## Open Questions

- 情境摘要的措辭模板要多少種情境分支？→ 先做 3 種（距退休 <5 年 / 5–15 年 / >15 年），上線後依回饋再擴充
- 風險屬性三選的描述文字要幾行？→ 每張卡 1 行主描述 + 1 行副描述
- B3 是否需要圖表 → 本次純文字，下版再加
