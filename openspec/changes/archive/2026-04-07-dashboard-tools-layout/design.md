## Context

目前 Dashboard 工具清單使用 `grid-cols-1 md:grid-cols-2 lg:grid-cols-3`，行動裝置上為單欄大卡片。每張卡片高度約 120px，7 個工具需要滾動約 900px 才能全部看完。卡片內有 icon、代碼、標題、說明文字、hover 箭頭，資訊密度偏低。

## Goals / Non-Goals

**Goals:**
- 在行動裝置上減少工具清單的滾動距離
- 提供三種視覺方案，讓用戶選擇偏好的排版
- 方案之間保持相同的互動行為（點擊跳轉）

**Non-Goals:**
- 不修改工具的資料（名稱、路由、說明不變）
- 不修改 StatCard 或其他共用元件
- 不加入新的工具項目

## Decisions

### 決策一：三種方案並列設計，選一實作

提供三種方案的完整規格，用戶確認後選一實作，而非同時實作切換功能（增加複雜度）。

**方案一：2 欄緊湊網格**
```
[icon] S1 財務現況輸入     [icon] S2 三桶金總覽
       輸入資產收入...            短中長期三桶...
[icon] A1 退休目標計算     [icon] A2 壓力測試
       計算所需資金...            Monte Carlo...
```
- grid-cols-2，卡片 p-3，icon 縮小至 w-7 h-7
- 說明文字縮減至 1 行（line-clamp-1）
- 行動裝置可見約 4 個工具不需滾動

**方案二：分區標題 + 單欄卡片**
```
── 共用工具 ──────────────
[S1] 財務現況輸入
[S2] 三桶金總覽
[S3] 通膨侵蝕模擬

── 退休前 ────────────────
[A1] 退休目標計算
...
```
- 依路由前綴 S/A/B 分組，各組有分區標題
- 卡片維持單欄但高度壓縮（p-2.5）
- 分區間距清晰，視覺層次更好

**方案四：列表樣式**
```
● [icon] 財務現況輸入  輸入資產、收入...  S1 →
● [icon] 三桶金總覽    短中長期三桶金...  S2 →
```
- 每列高度約 44px，7 個工具約 320px 全可見
- icon 16px，標題 13px，說明 11px 截短
- 最緊湊，適合快速瀏覽

### 決策二：保留 hover 效果
三種方案皆保留 `hover:border-blue-700` 的互動回饋，維持一致的操作體驗。

## Risks / Trade-offs

- **方案一**：說明文字 line-clamp-1 可能截斷重要資訊 → 可 hover 展開或接受截斷
- **方案二**：B 系列工具（B1-B4）目前不在 Dashboard 工具清單內，分區標題若加「退休後」分區會是空的 → 暫時只顯示 S 和 A 系列，或補入 B 系列工具
- **方案四**：說明文字極短，工具辨識度依賴標題文字 → 確保標題足夠描述性
