## Context

B2 `CashflowTimeline.tsx` 目前逐年計算退休後的現金流，已有「當年被動收入 = 勞保年金 + 勞退月領」邏輯與「提領率 = 提領需求 / 年初資產」。但缺乏「穩定收入 vs 支出」的摘要視角——使用者無法一眼看出穩定被動收入能覆蓋退休支出的比例，尤其是退休初期 60–64 歲的「空窗期」風險。

B1 已有單一的「被動收入覆蓋率 = 月被動收入 / 月支出」，但：
1. 只看退休那一年，不反映逐年變化
2. 沒有區分「空窗期 / 勞退期 / 全領期」三階段
3. 沒有圖形化呈現通膨侵蝕下的覆蓋率下滑

## Goals / Non-Goals

**Goals:**
- 讓使用者一眼看到「退休後穩定收入能撐住幾成支出」
- 揭露退休初期（勞退請領前）覆蓋率偏低的風險
- 將覆蓋率隨通膨下滑的自然現象「說明清楚」，避免誤讀
- 對齊課程 CH4「讓錢穩定流進來」的教學主軸
- 三桶金是資產面穩定性，穩定現金流是收入面穩定性，兩者在 APP 中互補呈現

**Non-Goals:**
- 不新增使用者輸入欄位（沿用 S1 既有資料：勞保/勞退/租金）
- 不修改 B2 的逐年提領邏輯（只加摘要區塊）
- 不處理「實質購買力衰退」獨立圖表（v1 選 B 方案加註說明即可）
- 不計入投資配息為「穩定收入」（課程中被視為有波動）
- 不支援「自訂穩定收入類別」（年金保險等未來擴充）

## Decisions

### D1：穩定收入定義

**決定**：穩定收入 = 勞保年金 + 勞退月領 + 租金收入，各自判斷請領年齡。

**理由**：
- 勞保/勞退是終身給付，最接近「穩定」定義
- 租金收入 S1 已有欄位，使用者若填了代表認為這是現金流來源；實務上空置風險在試算階段先不處理
- 投資配息波動大，排除以符合「穩定」原意
- 三桶金提領會消耗本金，不算「穩定現金流」

### D2：計算時機與通膨處理

**決定**：逐年迭代，支出套用通膨調整，穩定收入採「名目固定」（不通膨調整）。

**理由**：
- 勞保/勞退在實務上有微幅物價指數連動，但試算階段以「最保守假設」呈現，凸顯通膨風險
- 分母通膨、分子名目 → 覆蓋率會自然隨年齡下滑，這是「特性」不是 bug
- 需在圖表下加註解釋，避免使用者誤以為是計算錯誤

**公式**：
```
各年覆蓋率 = 當年穩定收入 / 當年通膨後月總支出 × 100%
  其中
    當年穩定收入 = (age ≥ 勞保請領年齡 ? 勞保年金 : 0)
                 + (age ≥ 勞退請領年齡 ? 勞退月領 : 0)
                 + 租金收入（退休期間持續計入）
    當年通膨後月總支出 = 月總支出 × (1 + 通膨率/100)^(age − 目前年齡)
```

### D3：三階段識別演算法

**決定**：依勞保/勞退請領年齡自動分段，不固定 60-64 / 65-69 / 70+。

**理由**：
- 每人的勞保/勞退請領年齡不同，使用者在 S1 自行設定
- 固定年齡段會誤導設定不同請領年齡的使用者

**演算法**：
```
pensionStart = min(勞保請領年齡, 勞退請領年齡)
allStart     = max(勞保請領年齡, 勞退請領年齡)
retireAge    = 退休年齡

階段：
  if retireAge < pensionStart
    → 空窗期: retireAge ~ pensionStart−1
    → 勞退/勞保期: pensionStart ~ allStart−1
    → 全領期: allStart ~ 預期壽命
  else if retireAge < allStart
    → 勞退/勞保期: retireAge ~ allStart−1
    → 全領期: allStart ~ 預期壽命
  else
    → 全領期: retireAge ~ 預期壽命
```

### D4：摘要卡三個數字的定義

**決定**：
- **最低覆蓋率** = `min(各年覆蓋率)`，標示最弱的年齡區間
- **平均覆蓋率** = `avg(各年覆蓋率)`，整個退休期間
- **全齡穩定覆蓋率** = 全領期各年覆蓋率的平均

**理由**：三個數字涵蓋「最壞情況 / 整體感受 / 長期穩態」三個視角。

### D5：顏色分級與警示

**決定**：
| 覆蓋率 | 顏色 | 意涵 |
|-------|------|------|
| ≥ 100% | 🔵 blue | 完全涵蓋，不需動本金 |
| 60–100% | 🟢 green | 健康 |
| 30–60% | 🟡 amber | 部分覆蓋，需靠投資提領補足 |
| < 30% | 🔴 red | 不足，高度依賴資產 |

**理由**：
- 對齊既有設計系統的 blue/green/amber/red 四色
- 30% / 60% 的斷點參考 B1 現有整體健康狀態判斷

### D6：通膨說明置放位置

**決定**：圖表下方加灰色小字一行，不獨立做折線圖。

**文案**：
> 💡 覆蓋率隨年齡下滑是因為勞保/勞退金額相對固定，但支出隨通膨上漲，晚年覆蓋率偏低屬正常現象。想提升晚年覆蓋率，可增加不動產租金或長期投資配置。

## Risks / Trade-offs

1. **租金收入穩定性**：實務上租金有空置風險，目前把它全額計入穩定收入，可能高估覆蓋率。v1 選擇簡化，未來可加開關「是否視為穩定」。

2. **勞保/勞退實際有物價連動**：現行假設「名目固定」是最保守做法。若使用者反映失真，未來可新增「勞保年通膨調整率」參數（預設 1%）。

3. **視覺密度**：B2 頁面已有逐年表格與提領率警示，新增摘要卡可能讓頁面變長。實作時需留意 StableCoverageCard 緊湊排版，避免擠壓原有資訊。

4. **階段判斷邊界**：若使用者退休年齡晚於兩個請領年齡（如 68 歲退休、67 歲請領勞退），只有「全領期」一段，帶狀圖只顯示一段——這是正確的，但需在 UI 上處理單段也不失衡。
