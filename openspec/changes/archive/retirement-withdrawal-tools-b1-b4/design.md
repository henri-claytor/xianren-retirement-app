## Context

Prototype 已完成 S1~S3、A1~A4 共 8 個工具，全部使用 localStorage 儲存 `FinancialSnapshot`，並透過 `calcSummary()` 計算三桶金數據。B 系列工具建立在相同資料基礎上，聚焦退休後的提領與現金流管理，對應課程 CH4。

目前 `FinancialSnapshot` 已有：勞保月退、勞退月退、月支出（三分類）、三桶金金額、可投資資產。B 系列所需的計算皆可由此推導，**不需擴充資料結構**。

## Goals / Non-Goals

**Goals:**
- B1：依三桶金現況模擬提領順序，計算各桶可支撐月數及建議每月提領金額
- B2：視覺化每年收入來源（勞保/勞退開始年齡 + 投資提領），以時間軸呈現
- B3：讓使用者設定三桶金警戒門檻，低於門檻顯示警示與補充建議
- B4：比對目前三桶金比例與建議比例，計算再平衡操作金額
- 全部工具沿用 S1 資料，自動帶入不需重新輸入

**Non-Goals:**
- 不實作稅務計算（資本利得、二代健保等）
- 不實作實際券商下單或提醒推播
- 不實作多帳戶分開管理
- 不新增後端 API（維持全前端 prototype）

## Decisions

**決策 1：B1 提領順序邏輯**
採「短→中→長」順序提領，符合三桶金設計原則（短期桶先動用，長期桶最後）。每月需求 = 月支出 − 被動收入（勞保 + 勞退）；若有正結餘則無需提領。各桶耗盡時間以線性計算（不含投資增長，保守估計）。

替代方案：依報酬率動態模擬（複雜度高，與 A2 Monte Carlo 重疊）→ 不採用。

**決策 2：B2 時間軸資料來源**
勞保月退/勞退月退的「開始領取年齡」需新增兩個輸入欄位（預設 65 歲），其餘沿用 S1 資料。時間軸以 `RetirementAge → ExpectedLifespan` 為範圍，每年一列。

**決策 3：B3 警戒水位儲存**
警戒門檻（各桶最低月數）以獨立 localStorage key `xianren_alert_config` 儲存，與主要財務資料分離，避免影響 `FinancialSnapshot` 結構。

**決策 4：B4 再平衡基準**
沿用 A3 的生命週期建議比例（依距退休年限分段），已退休者使用「已退休」段比例（短 15% / 中 45% / 長 40%）。

## Risks / Trade-offs

- **B1 耗盡時間是線性估算** → 不含投資增長，結果偏保守；已在介面說明此為最保守情境
- **勞保/勞退開始年齡為新增欄位** → 若使用者未填，預設 65 歲；對既有 localStorage 資料向下相容
- **B2 時間軸在手機版資訊密度高** → 採垂直捲動 + 折疊設計，年份可展開明細
